# Project Export: Workspace Services Roadmap & Strategy Hub

This file serves as a complete **data and code bundle** for the **Workspace Services Roadmap & Strategy Hub**. It consolidates the architecture, database schemas, Power Automate designs, deployment workarounds, and the **entire unified source code** (`index.aspx`) in one place so you can seamlessly resume this project in a new session.

---

## 1. Project Overview & Target Architecture

This application is a **highly aesthetic, dual-view strategic roadmap** designed for internal leaders of **Workspace Services** (Devices & Applications, Communication & Collaboration, Colleague Services, AI Enablement, and EST). 

*   **Frontend**: A single-file ASPX application hosted inside a SharePoint document library (`Site Assets`) and embedded on a Modern Page via an Iframe web part. It supports:
    *   **Items View**: Searchable, expandable strategic OKR cards aligned to Enterprise Initiatives and Strategic Objectives, displaying completion progress decimal scales (`0.0` to `1.0`).
    *   **Timeline View**: A horizontal Gantt/swimlane chart grouped dynamically by **Service Area** or **Strategic Objective** across a 4-quarter range (Q3 2026 to Q2 2027).
    *   **Change History Log**: Surfaced inside the detail drawer by querying SharePoint's native item version history REST API.
    *   **Theme**: Glassmorphism aesthetic supporting active toggles for Light and Slate Dark mode.
*   **Backend**: A SharePoint Online List acting as a secure database.
*   **Integrations**: Power Automate Flows that automatically generate custom, sequential Roadmap IDs (e.g. `IS-WSS-DA-1`, `IS-WSS-EST-5`) upon item creation.

---

## 2. Why index.html downloaded (The Solution)

**The Issue**: SharePoint Online modern pages enforce strict browser file handling. When an `.html` file is loaded in an iframe, SharePoint forces a download header (`Content-Disposition: attachment`), which stops the page from rendering in your iframe.

**The Solution**: We renamed the frontend to `index.aspx` and combined all HTML, CSS styling, and JavaScript logic into this **single unified file**.
*   SharePoint natively executes `.aspx` files in the browser.
*   Combining the CSS and JS inside the ASPX file avoids relative path issues or cross-site blocking, meaning you only need to upload **one file** to make the MVP work!

## 4. Power Automate ID Autogeneration Flow

Create an Automated Cloud Flow in Power Automate:

1.  **Trigger**: *When an item is created*
    *   Site Address: `[Your SharePoint Site URL]`
    *   List Name: `WorkspaceServicesRoadmap`
2.  **Action: Get items**:
    *   List Name: `WorkspaceServicesRoadmap`
    *   Filter Query (OData): `ServiceArea eq '@{triggerOutputs()?['body/ServiceArea/Value']}'`
3.  **Action: Compose (Increment Count)**:
    *   Expression: `add(length(outputs('Get_items')?['body/value']), 1)`
4.  **Action: Compose (Abbreviation Mapping)**:
    *   Use a switch/conditional action matching `ServiceArea`:
        *   `Devices and Applications` $\rightarrow$ `DA`
        *   `Communication and Collaboration` $\rightarrow$ `CC`
        *   `Colleague Services` $\rightarrow$ `CS`
        *   `AI Enablement` $\rightarrow$ `AI`
        *   `EST` $\rightarrow$ `EST`
5.  **Action: Update item**:
    *   ID: `@{triggerOutputs()?['body/ID']}`
    *   RoadmapID: `IS-WSS-@{outputs('AbbrevMapping')}-@{outputs('IncrementCount')}`

        function fetchItemHistory(item) {
            return new Promise((resolve) => {
                if (!isSharePoint) { resolve(item.VersionHistory || []); return; }
                const siteUrl = _spPageContextInfo.webAbsoluteUrl;
                const endPoint = `${siteUrl}/_api/web/lists/getbytitle('WorkspaceServicesRoadmap')/items(${item.Id})/versions`;
                fetch(endPoint, { headers: { "Accept": "application/json; odata=verbose" } })
                .then(res => res.json())
                .then(data => {
                    const versions = data.d.results;
                    if (!versions) { resolve([]); return; }
                    const logs = [];
                    for (let i = 0; i < Math.min(versions.length, 10); i++) {
                        const cur = versions[i];
                        const prev = versions[i+1];
                        let changes = [];
                        if (prev) {
                            if (cur.Title !== prev.Title) changes.push("Renamed title");
                            if (cur.Progress !== prev.Progress) changes.push(`Progress updated to ${parseFloat(cur.Progress || 0).toFixed(2)}`);
                            if (cur.Status !== prev.Status) changes.push(`Status changed to ${cur.Status}`);
                            if (cur.StartDate !== prev.StartDate || cur.EndDate !== prev.EndDate) changes.push("Timeline dates shifted");
                        } else { changes.push("Item registered."); }
                        logs.push({ version: cur.VersionLabel, date: new Date(cur.Created).toLocaleDateString(), author: cur.Editor.LookupValue || "System User", change: changes.join(", ") });
                    }
                    resolve(logs);
                })
                .catch(() => resolve(item.VersionHistory || []));
            });
        }
